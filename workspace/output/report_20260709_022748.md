# SAP BTP 비용 산정 보고서

- 작성일: 2026-07-09
- 대상 시스템: 직원 법인카드 비용정산 시스템
- CSP: SAP BTP
- 예상 사용자 수(기준): 4,000명 (비교 시나리오: 1,000 / 2,000 / 4,000 / 8,000명)
- 적용 환율: 1 EUR = 1,480 KRW (가정값, 기준일: 2026-07-09 — 사용자 확인 필요)

## 1. 요약

CSV 입력(직원 법인카드 비용정산 시스템, 예상 사용자 4,000명) 기준으로 SAP BTP 리소스(HANA Cloud, Cloud Foundry 애플리케이션 런타임, Object Store)의 사용량과 비용을 산정했습니다. 사용자 수가 증가할수록 Cloud Foundry 인스턴스 수와 스토리지(HANA/Object Store)가 선형적으로 증가하지만, **HANA Cloud는 티어(스토리지 용량) 단위로만 제공**되어 특정 사용자 수 구간을 넘으면 비용이 계단식으로 급증합니다. 4,000명 규모(CSV 기준)에서는 HANA storage 요구량(약 160GB)이 small 티어 한도(128GB)를 초과해 medium 티어(1,650 EUR/월)가 필요하며, 이는 전체 비용의 약 93%를 차지하는 핵심 비용 요인입니다.

## 2. 리소스별 예상 사용량

가정: 평일 일간 활성 사용자(Weekday DAU) = 전체 사용자의 15%, 동시접속률 = DAU의 10%

| 전체 사용자 수 | Weekday DAU | 동시접속자 수 | HANA storage 요구량(GB) | Object Store storage(GB) | CF 인스턴스 수(추정) |
|---|---|---|---|---|---|
| 1,000 | 150 | 15 | 40 | 25 | 4 |
| 2,000 | 300 | 30 | 80 | 50 | 4 |
| 4,000 (CSV 기준) | 600 | 60 | 160 | 100 | 4 |
| 8,000 | 1,200 | 120 | 320 | 200 | 8 |

※ HANA storage·Object Store storage는 CSV 기준값(4,000명 → HANA 160GB, Object 100GB)을 사용자 수에 비례 스케일링한 추정치입니다.
※ CF 인스턴스 수 = max(4, ceil(전체 사용자수/1,000)) — 앱 4종(portal/service/file uploader/logger) 최소 1개씩 기준.

⚠ 4,000명 시점 HANA storage 요구량(160GB)이 small 티어 한도(128GB)를 초과 → medium 티어 필요 (비용 급증 지점).
⚠⚠ 8,000명 시점 HANA storage 요구량(320GB)이 medium 티어 한도(256GB)마저 초과 → 캐시에 없는 상위 티어 또는 스케일 아웃 구성이 필요하므로 **SAP Discovery Center에서 large/전용 HANA Cloud 티어 확인 필요** (본 보고서에서는 medium 비용으로 임시 표기, 실제로는 과소 산정 가능).

**비용 비율 (4,000명 기준)**: HANA 92.6% / CF 7.3% / Object Store 0.1%

## 6. 가정 목록

- 입력 방식: CSV (/input/BTP Spec.txt)
- 전체 사용자 수: 1,000 / 2,000 / 4,000(CSV 기준) / 8,000명 시나리오 비교
- 평일 일간 활성 사용자 수(Weekday DAU): 전체 사용자의 15% 가정
- 동시접속률: Weekday DAU의 10% 가정
- HANA storage 요구량: 4,000명 기준 160GB(CSV 120~200GB 중간값)를 사용자 수에 선형 비례 스케일링
- Object Store storage: 4,000명 기준 100GB(CSV 값)를 사용자 수에 선형 비례 스케일링
- CF 인스턴스 수: max(4, ceil(전체 사용자수/1,000)), 티어는 cf.standard.small(1vCPU/2GB) 기본 가정
- 월 사용 시간: 720시간(24시간×30일, 상시 가동 가정)
- 환율: 1 EUR = 1,480 KRW (가정값 — **실제 환율로 교체 확인 필요**)
